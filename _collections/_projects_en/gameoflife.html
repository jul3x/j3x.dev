An implementation of the popular cellular automaton "Conway's Game of Life" written in x86_64 Assembly (NASM syntax).

How does it work? A grid of live or dead cells with dimensions n x m. The game proceeds iteratively. In each new iteration, every cell changes its state or remains the same.

- If a cell is dead and has exactly three live neighbors, it comes to life.
- If a cell is alive and has two or three live neighbors, it stays alive.
- Otherwise, it dies.

These three simple rules allow for the construction of very advanced and interesting patterns, which have been the subject of many studies worldwide over the years. "The Game of Life" is Turing complete (in simple terms: it can solve the same set of problems as a computer). For example, this allows for building a computer with a monitor or playing the Game of Life within the Game of Life.
